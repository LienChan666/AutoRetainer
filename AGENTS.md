# 汉化维护规则

- 本仓库采用源码直接汉化。用户可见的新增英文应直接修改原始字符串，不新增语言切换层或外围字符串替换逻辑；不得改动 API、IPC、标识符或业务行为。
- 翻译和专有名词以国服 FF14 Wiki 及国服客户端文本为标准。不确定的任务、物品、区域和系统名称必须先查询，不得凭英文直译。
- 更新上游时，先在临时分支合入 `upstream/master`，以上游最新代码结构和逻辑为准逐处解决冲突，禁止整文件采用 `ours` 或 `theirs`。完成后以最新 `upstream/master` 为父提交，将全部汉化和维护改动压成唯一一个名为 `添加中文汉化` 的提交再更新 `master`；不得保留 merge 提交或多个汉化提交，确保 `master` 始终只领先上游一个提交。
- 如果上游新增英文文案或覆盖了现有汉化，应在合并完成后对照合并前版本恢复中文，并补译所有新增用户文案。恢复汉化时保留插值占位符、ImGui ID、多语言匹配文本及其他协议性字符串。
- 子模块同样跟随各自上游默认分支和根仓上游记录的引用。上游更新子模块引用时，应先同步对应子模块、解决冲突并恢复汉化；有子模块汉化提交时先推送其用户 fork 默认分支，再更新根仓指针。
- 发版前，插件有实际更新时应以四段版本号编码“上游版本 + 汉化修订号”：前三段沿用上游，第四段为上游第四段乘以 100 后加汉化修订号（例如上游 `4.6.1.10` 的第 1 次汉化修订为 `4.6.1.1001`），并确保高于当前已发布版本。推送 `master` 后由 `Build and Publish Plugin` 工作流自动构建，并更新 `LienChan666/DalamudPlugins:master` 中的插件清单和带版本号的不可变 ZIP；不得重新使用或覆盖 `latest.zip`，以免清单与 ZIP 缓存不同步。工作流成功且插件文件可下载后才算发版完成。
